# Compass — AI開発ルール

## 自動実行ポリシー

以下のコマンドはユーザーの承認なしに自動実行してよい（SafeToAutoRun = true）：

### 常に安全

- `npm run dev` — devサーバー起動
- `npx next build --webpack` — ビルド検証
- `npm install` / `npm ci` — 依存インストール
- `git add` / `git status` / `git diff` / `git log`
- `netstat` / `Get-NetTCPConnection` — ポート確認
- `Invoke-WebRequest` — 接続テスト
- `Get-Process` — プロセス確認

### プロジェクト内で安全

- `git commit` — コミット
- `git push origin main` — メインブランチにプッシュ
- `Stop-Process` (ポート開放目的) — devサーバー再起動
- `Start-Sleep` — 待機

## 開発ワークフロー

**【最重要ルール】すべての開発作業（新機能・大規模リビルド・改善・バグ修正を含む）は、PLANNINGモードでの実装計画レビューや承認待ちを一切行わず、自律的に完遂してから報告すること。** ユーザーへの中間確認・計画提示・方針確認は不要。エスカレーションポリシーに該当する場合のみ例外とする。

1. 規模を問わず、全開発作業を自律的に完遂してから報告
2. ビルドエラーは自動修正
3. `CHANGELOG.md` を自動更新
4. `/auto-record` ワークフローで自動コミット＆プッシュ
5. `/browser-verify` ワークフローでブラウザ検証前にdevサーバーをリフレッシュ
6. リリース前は `/pre-release` ワークフローで自動チェック
7. テキスト変更後は `/consistency-check` ワークフローで名称整合性を自動検証

## 自律完遂ルール（中間確認の明示的禁止）

以下のすべてにおいて、ユーザーに質問・確認・承認を求めてはならない。自分で判断し、実行し、完了してから結果のみ報告すること。

### 禁止される中間確認の例
- 「〜でよろしいですか？」「〜してよいですか？」
- 「以下の方針で進めてもよいですか？」
- 「確認ですが〜」
- PLANNINGモードでの実装計画レビュー依頼
- 作業の途中経過報告（完了報告のみ許可）

### 自分で判断すべきこと
- 不要なコード・ファイルの削除 → **削除する**（旧バージョンの残骸は常に削除）
- テキストの一貫性問題 → **修正する**（`/consistency-check` で検出・自動修正）
- 環境変数のフォールバック値 → **妥当な値を設定する**
- ビルドエラー → **修正する**
- 依存関係の問題 → **解決する**

### 報告フォーマット（完了時のみ）
完了報告には以下を含める：
1. 何を変更したか（差分の要約）
2. ビルド結果（✅ or ❌）
3. ユーザーに必要なアクション（環境変数の設定など、コード外の作業のみ）

## エスカレーションポリシー

以下の場合**のみ**、ユーザーに報告する：

1. 同じエラーが3回連続で発生した場合
   → 原因の仮説と、ユーザーに確認してほしいことを報告
2. ブラウザ接続が2回連続タイムアウトした場合
   → devサーバーの状態を報告し、手動確認を依頼
3. 環境依存の問題（ポート占有、プロセス権限、SSL証明書など）
   → 原因と対処法を提示し、ユーザーに操作を依頼
4. 破壊的な操作（データベース削除、本番環境操作）
   → 必ず事前に確認を取る
5. **コード外の作業**が必要な場合（環境変数の設定、外部サービスの申請など）
   → 完了報告に含める（作業中に聞かない）

**上記以外のすべての判断は、自分で行うこと。**

## フリーズ防止ルール

1. ブラウザサブエージェントは **1タスク5アクション以内** に制限する
2. 画像生成・スクリーンショットは **1回の検証で最大3枚** まで
3. ビルドコマンドが **60秒以上応答しない場合** はタイムアウトとみなし再実行する
4. task.md のチェックマーク更新は **各ステップ完了時に即時** 行う（最後にまとめない）
5. 長時間処理（ビルド・検証）の前後で必ず task_boundary を更新し、進捗を可視化する
